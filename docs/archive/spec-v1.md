## 1. 产品概述

### 1.1 产品定位
跨平台桌面 Markdown 阅读器，以工作区方式浏览本地目录中的 Markdown 文件，提供高质量的渲染体验和高效的文件导航。

### 1.2 目标用户
- 技术文档写作者
- 知识管理用户
- 需要本地查看大量 Markdown 文件的开发者

### 1.3 核心价值
- 轻量快速：启动快、渲染流畅
- 专注阅读：无编辑干扰，专注内容浏览
- 本地优先：数据全部存于本地，保护隐私

## 2. 功能需求

### 2.1 文件浏览
- 以目录为工作区，递归展示文件树（惰性加载，展开时才读取子目录）
- 隐藏文件/目录（`.` 开头）灰色显示但不排除
- 内置忽略目录：`.git`、`node_modules`、`__pycache__`、`.DS_Store`（后端硬编码）
- 设置面板支持编辑忽略列表和 Markdown 扩展名列表（配置持久化保存，但后端目录扫描暂未应用自定义过滤规则）
- 配置变更后触发文件树刷新

### 2.2 多标签管理
- 顶部标签栏，每个打开的文件对应一个标签
- 点击标签切换当前阅读文件
- 可关闭标签（关闭按钮 / 鼠标中键）
- 文件外部修改后标签短暂显示刷新提示
- 文件内容惰性加载（切换到标签时才渲染）

### 2.3 Markdown 渲染
- 支持 GFM 全套：表格、任务列表、删除线、脚注、自动链接
- 支持数学公式：行内 `$...$` 和块级 `$$...$$`
- 支持 Mermaid 图表：` ```mermaid ` 代码块渲染
- 支持代码高亮：自动语言识别 + 语法着色
- 大纲面板：右侧显示当前文件标题层级（H1-H6）
- 点击大纲标题跳转至对应位置
- 滚动时高亮当前可视区域的大纲项

### 2.4 搜索
- 文件搜索：按文件名搜索并打开（快捷键 Ctrl+P），在已加载的文件树数据中匹配
- 全局内容搜索：在工作区文本文件中搜索文本（快捷键 Ctrl+Shift+F）
- 搜索结果渐进式显示（后端逐文件扫描，通过事件推送进度）

### 2.5 主题
- 支持暗色/亮色双主题
- 默认跟随系统主题偏好
- 可手动切换主题并持久化
- system 模式下实时响应操作系统主题变化

### 2.6 状态恢复
- 记录最近打开的文件列表
- 记录最近访问的目录列表
- 记录每个文件的阅读位置（滚动位置）
- 启动时可恢复上次完整阅读状态：工作区目录 + 已打开文件 + 各文件滚动位置

### 2.7 文件监控
- 自动监控所有已打开文件的磁盘变更
- 文件被外部程序修改后自动重新渲染
- 刷新后保持当前滚动位置

### 2.8 链接与图片
- 内部 `.md` 链接：在应用内新标签中打开
- 外部链接（http/https）：调用系统默认浏览器打开
- 本地图片：支持相对路径引用，正常显示

### 2.9 设置面板
- 主题切换（System / Light / Dark）
- 忽略列表编辑（一行一个条目）
- Markdown 扩展名配置（支持无扩展名的文本文件）

### 2.10 面板布局
- 三栏布局：侧边栏（文件树+搜索） + 主内容区 + 大纲面板
- 侧边栏与大纲面板宽度可通过拖拽调节
- 面板宽度持久化保存，启动时恢复
- 侧边栏与大纲可分别切换显示/隐藏

### 2.11 CLI 命令行参数
Tauri 框架内建版本号查询，项目暂未实现自定义 CLI 参数解析。

> 注：Electron 时代曾实现 `-v` / `-h` 参数，迁移至 Tauri 后待补充。

## 3. 非功能需求

### 3.1 平台兼容性
- 支持 Windows、macOS、Linux 三平台
- 当前打包配置：Windows NSIS
- 历史配置（Electron 时代）：Windows NSIS、macOS DMG、Linux AppImage + deb

### 3.2 性能
- 冷启动时间 ≤ 3 秒（常规硬件）
- 10MB 以内 Markdown 文件流畅渲染
- 文件搜索结果首屏响应 ≤ 500ms（千级文件目录）

### 3.3 安全
- 自定义协议 `local-file://` 仅加载本地图片资源，最小权限原则
- 前端与后端通过 Tauri IPC 通信，前端无文件系统直接访问能力
- 外部链接统一经 shell 插件打开，不直接在渲染进程跳转
- Tauri 能力配置（capabilities）精确控制前端可调用的后端接口

### 3.4 可维护性
- Rust 后端与 React 前端分层清晰，IPC 命令契约明确
- 单元测试覆盖核心业务逻辑
- E2E 测试覆盖主要用户路径

## 4. 技术选型

| 层 | 选型 |
|---|------|
| 桌面框架 | Tauri 2 |
| 后端语言 | Rust |
| 构建工具 | Vite + Tauri CLI |
| 打包工具 | Tauri bundler |
| 前端框架 | React 19 + TypeScript |
| UI | Tailwind CSS（自研组件） |
| 状态管理 | zustand |
| Markdown 渲染 | react-markdown + remark-gfm + remark-math + rehype-katex + rehype-highlight + rehype-raw |
| Mermaid 图表 | mermaid（动态 import） |
| 代码高亮 | highlight.js（按需注册 15 种常见语言） |
| IPC 通信 | Tauri invoke / emit + 官方插件（fs / dialog / shell） |
| 本地资源 | 自定义 protocol `local-file://` |
| 持久化 | localStorage |
| 测试框架 | Vitest（单元）+ Playwright（E2E） |

## 5. 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| Ctrl+Shift+O | 打开文件夹 |
| Ctrl+P | 文件搜索 |
| Ctrl+Shift+F | 全局内容搜索 |
| Ctrl+B | 切换文件树显示 |
| Ctrl+Shift+L | 切换大纲面板 |
| Ctrl+W | 关闭当前标签 |
| Ctrl+Tab | 下一个标签 |
| Ctrl+Shift+Tab | 上一个标签 |
| Ctrl+, | 打开设置 |

> 注：快捷键通过渲染进程 Hook（`useKeyboardShortcuts`）在 DOM 级实现，无需原生菜单注册。具体实现见 [architecture.md](../architecture.md)。
